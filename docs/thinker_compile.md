本文档旨在详细介绍如何在x86 Linux环境下编译Thinker框架，并提供相关编译参数的说明和使用示例。  
通过本文档，开发者可以了解各个编译选项的作用，以及如何根据实际需求进行配置。

## 编译脚本
thinker的根目录下运行
```Shell
sh scripts/x86_linux.sh
```

## 编译参数说明
### 基本参数
| 参数 | 类型 | 默认值 | 描述 |
|-------|-------|-------|-------|
| CMAKE_BUILD_TYPE | 字符串 | Debug | 设置构建类型，可选值为 Debug 或 Release |
| ARCH | 字符串 | x86_64 | 指定目标架构，这里是x86_64，表示64位x86架构|

注意：仿真平台仅在x86_64 linux系统下完成验证测试
### 功能模块配置
| 参数 | 类型 | 默认值 | 描述 |
|-------|-------|-------|-------|
| THINKER_SHARED_LIB | 布尔值 | ON | 启用或禁用共享库的构建。ON表示生成动态链接库（.so文件），OFF表示生成静态库（.a文件） | 
| THINKER_PROFILE | 布尔值 | ON | 启用或禁用性能测试功能。ON表示启用，便于性能分析 | 
| THINKER_RESULT_DUMP | 布尔值 | OFF | 启用或禁用中间结果打印。OFF表示关闭，减少调试输出 | 
| THINKER_RESULT_CRC_PRINT | 布尔值 | OFF | 启用或禁用中间结果CRC打印功能。OFF表示关闭，减少输出信息 | 
| THINKER_RESOURCE_CRC_CHECK | 布尔值 | OFF | 启用或禁用资源CRC检查功能。OFF表示关闭，禁用校验功能 | 
| THINKER_CHECK_PLATFORM | 布尔值 | ON | 启用或禁用平台检查功能。ON表示启用，确保资源与平台相互匹配 |

注意: 芯片平台无法使用THINKER_RESULT_DUMP功能（缺少文件系统），可通过打印中间结果CRC来对比结果一致性
### 平台配置
| 参数 | 类型 | 默认值 | 描述 |
|-------|-------|-------|-------|
| THINKER_USE_VENUS | 布尔值 | OFF | 启用或禁用Venus平台的配置。OFF表示不使用 |
| THINKER_USE_ARCS | 布尔值 | OFF |启用或禁用ArCS平台的配置。OFF表示不使用 |
| THINKER_USE_VENUSA | 布尔值 | ON | 启用或禁用VenusA平台的配置。ON表示使用 |
| THINKER_USE_MOSS | 布尔值 | OFF | 启用或禁用Moss平台的配置。OFF表示不使用 |

注意：平台必须是三选一，THINKER_USE_MOSS仅在venus平台有效，适用于定制的大算子加速计算
## 注意事项
* 环境依赖：确保已经安装了所需的开发工具和依赖项，如CMake 3.20.1或更高版本，gcc 4.8.5或更高版本。可能需要手动指定这些工具的路径
* 模块选择：根据实际需求选择启用或禁用相关平台和功能模块，避免不必要的资源消耗
* 构建类型：根据开发阶段选择合适的构建类型。Debug模式适合开发和调试，Release模式适合正式发布
* 并行编译：脚本中使用了-j16参数，表示使用16个并行进程进行编译，加快编译速度。可以根据实际CPU核心数调整该值
* 编译结果：thinker库和测试用例存放到根目录bin文件夹下
